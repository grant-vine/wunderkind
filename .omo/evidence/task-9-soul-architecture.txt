Key evidence captured:
- Current retained topology is six personas only: product-wunderkind, fullstack-wunderkind, marketing-wunderkind, creative-director, ciso, legal-counsel.
- Current agent sources still embed static personality prose directly in prompt strings.
- src/index.ts already performs runtime system-prompt injection for docs output and resolved config, making runtime SOUL overlays the natural extension point.
- commands/docs-index.md already treats docs files as managed per-agent lanes, which supports appending persona-specific durable pointers after docs refresh.
- src/agents/docs-config.ts shows legal-counsel is not docs-eligible today, so /docs-index SOUL updates currently apply to the other retained docs owners only.

Decisions fixed in the architecture document:
- SOUL location convention: .wunderkind/souls/<agent-key>.md
- SOUL format: markdown v1 file with fixed Customization and Durable Knowledge sections
- Injection mechanism: runtime in src/index.ts after resolved runtime context and before native agent catalog
- Neutral rule: no SOUL file means no SOUL payload and no build-time prompt bloat
- Init flow: opt-in confirm, retained-persona multiselect, four exact framing questions per selected persona
- Learning-event updates: append-only durable-knowledge entries with project-relative pointers and dedup rule

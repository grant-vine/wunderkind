Plan: memory-redesign.md
Findings:
- Transform wunderkind's memory layer into a fully pluggable, multi-datastore "Ultimate Memory System" (UMS).
- Xenova/e5-small-v2 (Microsoft, 32MB int8, 384d, MTEB 59.9) as default embedding model.
- voyage-4-nano (2048d) as premium option via onnx-community/voyage-4-nano-ONNX.
- Hybrid search via RRF (k=60), weight_fts=1.3, weight_vec=1.0.
- SQLite hybrid RRF search (FTS5 + sqlite-vec).
- MiniSearch for FileAdapter BM25 index.
- New DuckDBAdapter ("timestream") for hot/warm/cold tiering.
- New CompositeAdapter for fan-out writes and RRF read merge.
- TTL policies: procedural (null), factual (90d), contextual (7d), observational (3d).
- Auto-pin heuristic: access_count >= 5 AND age_days >= 3, or keyword-based (manual pinnedReason is sticky).
- Contradiction detection: 3-gram Jaccard > 0.6.
- LLM consolidation: phi4-mini:3.8b via Ollama, count-based trigger (N=100), structured diff output.
- Storage budget: limitMb (default 500MB), LRU eviction excludes pinned.
- Memory diagram command using Mermaid.
- mem0 deprecated due to latency (380ms/write) and incompatibility with bulk writes.
- Story-based recall accuracy testbench (bun run bench:recall) using LongMemEval and LoCoMo datasets.

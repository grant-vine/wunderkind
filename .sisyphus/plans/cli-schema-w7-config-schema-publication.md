# CLI Schema W7 — Config Schema Publication

## Objective

Publish a formal JSON Schema for Wunderkind's configuration files and integrate it into the generated config via the `$schema` field.

## Scope

- Define the Wunderkind config schema in JSON.
- Implement `$schema` generation in the config-manager.
- Establish a stable and immutable GitHub URL strategy for schema publication.

## Depends On

- W2 — Config Contract Split (for final schema structure)

## Files in Scope

- `schemas/wunderkind.config.schema.json` (New file)
- `src/cli/config-manager/index.ts`
- `package.json` (Export/Include asset)

## Product Decisions / Frozen Contract

- **Schema URL**:
  - Latest Alias: Raw GitHub URL from the `main` branch.
  - Immutable Tag: Raw GitHub URL from the specific release tag.
- Config files generated by the installer must include a top-level `$schema` field pointing to the "latest" alias.
- The schema must be **scope-aware**. It must clearly distinguish between fields owned by Global scope and fields owned by Project scope as defined in W2.
- Do NOT describe a single merged shape that implies project-fields are valid in global files.

## Deliverables

- Scope-aware schema definition(s) that do not imply project-only fields are valid in global config files.
- `writeWunderkindConfig()` updated to include `$schema`.
- Published schema URL strategy documented in `README.md`.

## Task Breakdown

### Task W7.1 — Author the JSON Schema

- **Action:** Create `schemas/wunderkind.config.schema.json`.
- **Action:** Define scope-aware schema structure for Global and Project config ownership as frozen in W2.
- **Action:** Add validation for `region`, `industry`, `primaryRegulation`, and personality enums.

### Task W7.2 — Update Config Writer with `$schema`

- **Action:** In `src/cli/config-manager/index.ts`, update the config template used by `writeWunderkindConfig()`.
- **Action:** Add the `$schema` field at the top, pointing to the stable GitHub URL.

### Task W7.3 — Document Schema Usage

- **Action:** Update `README.md` to explain how to use the schema for manual configuration.
- **Action:** Include instructions for both "latest" and tagged URLs.
